# 🏆 金条管家 v1.1 - 保姆级教程

一个专为《明日之后》游戏设计的自动化金条监控与管理工具。它通过先进的OCR技术实时读取游戏内的金条数量，并将数据记录到本地数据库，通过一个美观的Web界面进行展示和分析。

## ✨ 核心功能

-   👁️ **精准OCR识别**: 采用先进的 **EasyOCR** 引擎，自动识别游戏中的金条数字，支持 **GPU加速**。
-   🎯 **聚焦窗口扫描**: 只扫描当前用户操作的游戏窗口，完美支持多开，想扫哪个点哪个。
-   💾 **智能数据记录**: 仅在金条数量变化时记录，有效避免数据冗余，并计算每次变化的差值。
-   🚀 **现代化GUI启动器**: 通过图形化界面一键启停所有服务，实时查看运行日志，管理方便。
-   🤫 **一键后台运行**: 提供 `.vbs` 脚本，可将程序 **完全隐藏** 在后台静默运行，无任何弹窗或图标。
-   📊 **实时Web面板**: 在本地网页上查看所有账号的最新金条数据、周变化、总资产等信息，并支持排序。
-   📈 **完整历史追溯**: 提供独立的页面，可查看所有历史记录，并按账号进行筛选。
-   🧠 **自动内存管理**: 扫描器内置内存监控和 **引擎自动重建** 机制，确保长时间稳定运行不卡顿。
-   🔊 **即时音频反馈**: 通过不同的提示音反馈操作结果（如成功、失败、未找到窗口），交互体验更佳。
-   ⚙️ **高度灵活配置**: 通过简单的 `config.json` 文件即可自定义热键和多分辨率下的扫描坐标。
-   📂 **分组与团队管理**: 新增分组功能，可将不同账号归类管理，方便团队金农。
-   ✅ **周常任务追踪**: 在主面板上直观地追踪每个账号的周常任务完成情况。
-   🔒 **小黑屋 (封禁功能)**: 可将账号临时封禁，封禁期间的账号会自动移至表格底部，到期后自动恢复。

## 🛠️ 环境准备 (新手必看)

本工具需要Python环境和一些依赖库才能运行。如果您是第一次接触，请严格按照以下步骤操作。

### 1. 安装 Python

-   **下载**: 前往 [Python官网](https://www.python.org/downloads/) 下载最新版的Python（推荐 3.9+）。
-   **安装**:
    1.  运行下载好的安装包。
    2.  **极其重要**: 在安装界面的第一步，**务必勾选 `Add Python to PATH`** 这个选项！
    3.  点击 `Install Now`，等待安装完成即可。


### 2. (可选) 安装CUDA以启用GPU加速

如果您的电脑有NVIDIA显卡，强烈建议安装CUDA，可以极大地提升OCR识别速度。

-   **检查显卡**: 确保您有NVIDIA显卡。
-   **安装PyTorch**: 打开命令行工具（在开始菜单搜索`cmd`），运行以下命令来安装与CUDA兼容的PyTorch。这是EasyOCR使用GPU所必需的。
    ```bash
    pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
    ```
    > 如果您的CUDA版本不同，请访问 [PyTorch官网](https://pytorch.org/get-started/locally/) 获取对应的安装命令。如果安装失败或没有N卡，程序会自动回退到CPU模式，不影响使用。

## ⚙️ 三步完成配置

在运行程序前，您只需要完成三件小事：安装依赖、配置坐标和热键。

### 第一步：安装项目依赖

将本工具所有文件解压到一个固定的文件夹。

1.  在该文件夹的地址栏输入 `cmd` 然后按回车，会弹出一个黑色的命令行窗口。
2.  在窗口中，复制并粘贴以下命令，然后按回车：
    ```bash
    pip install -r requirements.txt
    ```
3.  等待所有依赖包自动安装完成。如果出现红色错误，通常是网络问题，多试几次即可。
    > **💡 提示**: 如果反复失败，可以尝试使用国内镜像源加速下载。将命令替换为：
    > `pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`

### 第二步：配置坐标

您需要告诉程序去游戏画面的哪个位置截图识别金条。

1.  **运行坐标获取工具**: 双击运行文件夹中的 `get_mouse_coords.py` 文件。
2.  **打开游戏**: 将游戏窗口调整到您平时玩游戏的大小。
3.  **获取坐标**:
    -   将鼠标指针移动到金条数字区域的 **左上角**。
    -   查看坐标工具窗口中显示的 **“窗口内相对坐标 (x, y)”**，并记录下来。
    -   目测一下金条数字区域的 **宽度(width)** 和 **高度(height)**。估算即可，稍微大一点没关系。
4.  **填入配置文件**:
    -   用记事本打开 `config.json` 文件。
    -   找到与您游戏窗口分辨率最接近的条目，或者直接修改 `"default"` 条目。分辨率可以在游戏设置里查看。
    -   将刚才记录的四个值填入 `gold_region` 字段，格式为 `[x, y, width, height]`。

    **示例**:
    假设您的游戏分辨率是 `1920x1080`，获取的坐标是 `x=850, y=850`，估算的宽高是 `width=150, height=50`。
    那么您需要在 `config.json` 中这样填写：
    ```json
    {
      "hotkey": "f9",
      "use_gpu": true,
      "resolutions": {
        "1920x1080": {
          "scan_profiles": [
            {
              "name": "默认",
              "pixel_check": null,
              "gold_region": [ 850, 850, 150, 50 ]
            }
          ]
        }
      }
    }
    ```
    > **提示**: 您可以为多种不同的分辨率配置坐标，程序会自动匹配当前窗口的分辨率。

### 第三步：配置热键

您可以自定义触发扫描的快捷键。

-   打开 `config.json` 文件。
-   修改 `hotkey` 字段的值。默认为 `"f9"`。
-   您可以设置为单个按键如 `"f10"`，或组合键如 `"ctrl+shift+s"`。

### ⭐ 高级配置：上下文匹配 (选读)

为了应对在不同游戏界面（如“背包”和“交易所”）金条位置不一致的问题，v1.1版本引入了强大的 **上下文匹配** 功能。它允许您为同一个分辨率设置多个扫描配置（`scan_profiles`），程序在扫描时会自动判断当前处于哪个界面，并使用对应的坐标。

**工作原理：**

1.  **检查上下文**: 按下热键后，程序会先根据您配置的 `context_check` 区域进行一次小范围截图和OCR。
2.  **匹配关键字**: 程序会检查识别出的文本是否包含您设置的 `keywords` (如“背包”、“交易”)。
3.  **选择配置**: 如果匹配成功，则使用该配置档案 (`profile`) 中的 `gold_region` 来扫描金条；如果所有配置都未匹配，则使用不带 `context_check` 的那个“默认”配置。

**如何配置：**

假设在 `1920x1080` 分辨率下：
-   在**背包**界面，金条坐标是 `[850, 850, 150, 50]`。
-   在**交易所**界面，金条坐标是 `[860, 860, 150, 50]`。

您可以这样配置 `config.json`:

```json
{
  "hotkey": "f9",
  "resolutions": {
    "1920x1080": {
      "scan_profiles": [
        {
          "name": "背包界面",
          "context_check": {
            "region": [50, 150, 100, 50],
            "keywords": ["资源包", "背包"]
          },
          "gold_region": [850, 850, 150, 50]
        },
        {
          "name": "交易所界面",
          "context_check": {
            "region": [50, 100, 100, 50],
            "keywords": ["交易", "市场"]
          },
          "gold_region": [860, 860, 150, 50]
        },
        {
          "name": "默认",
          "context_check": null,
          "gold_region": [850, 850, 150, 50]
        }
      ]
    }
  }
}
```

**字段说明:**

-   `"name"`: (可选) 给这个配置起个名字，方便自己识别。
-   `"context_check"`: 上下文检查配置。
    -   `"region"`: `[x, y, width, height]`，一个包含关键字的小区域坐标。**用坐标工具获取**。
    -   `"keywords"`: 一个包含多个关键字的列表。只要识别出的文本包含**任意一个**关键字，即视为匹配成功。
-   `"context_check": null`: 这个配置是**默认的备用配置**。当其他所有带 `context_check` 的配置都匹配失败时，程序会使用这个。**强烈建议保留一个**。

通过这个功能，您可以为各种不同的界面（邮件、配方、排行榜等）添加专属的扫描配置，实现全场景的精准识别。

## ▶️ 如何运行

我们提供两种启动方式，请根据您的喜好选择。

### 方式一：使用图形化启动器 (推荐)

这是最简单、最直观的方式，适合绝大多数用户。

-   **启动**: 直接双击运行 `modern_launcher.py` 文件。
-   **操作**:
    -   **[🚀 一键启动]**: 点击此按钮，会自动启动后台Web服务器和扫描器。
    -   **[🛑 一键停止]**: 安全地关闭所有正在运行的服务。
    -   **[🔄 重启扫描器]**: 当您感觉扫描器卡顿时，用它来重启，无需关闭Web服务。
    -   **[🌐 打开面板]**: 在浏览器中打开数据显示面板。
    -   **[🎯 坐标工具]**: 快速启动坐标获取工具。
-   **监控**:
    -   **服务状态**: 界面会实时显示`服务器`和`扫描器`的运行状态（运行中/已停止）。
    -   **实时日志**: 界面下方的文本框会滚动显示最新的运行日志，方便您监控程序动态。

### 方式二：一键后台静默运行 (进阶)

如果您不希望看到任何界面，只想让它在后台默默工作，请使用此方式。

-   **启动**: 直接双击运行 `启动_隐藏版.vbs` 文件。
-   **效果**: 程序将 **完全在后台运行**，不会有任何窗口或任务栏图标。
-   **如何使用**:
    1.  双击 `.vbs` 文件启动。
    2.  打开游戏，按您设置的热键即可扫描。
    3.  手动在浏览器访问 `http://localhost:8080` 查看数据。
-   **如何关闭**:
    -   打开 **任务管理器 (Ctrl+Shift+Esc)**。
    -   在“详细信息”标签页中，找到名为 `python.exe` 的进程，根据“命令行”列找到本工具对应的进程，然后结束它。
    -   或者，直接运行启动器 `modern_launcher.py`，点击 **[🛑 一键停止]** 按钮，它会帮忙关闭后台进程。

## 🎮 日常使用流程

1.  **启动程序**: 选择上述任一方式启动金条管家。
2.  **打开游戏**: 正常启动《明日之后》客户端。
3.  **扫描金条**:
    -   点击您想扫描的游戏窗口（使其成为当前聚焦窗口）。
    -   按下您在 `config.json` 中设置的热键（默认为 `F9`）。
    -   您会听到提示音，代表扫描和数据提交的结果。
4.  **查看数据**:
    -   在浏览器打开 `http://localhost:8080`。
    -   数据面板会显示所有扫描过的账号的最新金条数，并自动刷新。
5.  **管理账号**:
    -   在主面板上，您可以直接修改账号的等级、备注，以及勾选周常任务。
    -   在“分组管理”页面，您可以创建分组，并将账号拖拽或移动到不同分组中。

## 🐛 故障排除 (FAQ)

**Q1: 程序闪退，或者双击 `.py` 文件没反应？**
**A1:**
-   **最大可能**: 您没有通过命令行 `pip install -r requirements.txt` 安装依赖。请严格按照 **[第一步：安装项目依赖](#第一步安装项目依赖)** 操作。
-   **其次可能**: 您安装Python时忘记勾选 `Add Python to PATH`。请卸载Python后重新安装，并务必勾选此项。
-   **查看日志**: 如果使用启动器，底部的日志区域会显示详细的错误信息，可以根据信息判断问题。

**Q2: 按下热键后没反应，或者听到错误提示音？**
**A2:**
-   **检查扫描器状态**: 如果使用启动器，请确认“扫描器”的状态为“运行中”。
-   **检查日志**: 启动器的日志会告诉您发生了什么。常见的日志有：
    -   `当前没有聚焦的明日之后游戏窗口`: 您需要先用鼠标点击一下游戏窗口再按热键。
    -   `EasyOCR识别失败`: 坐标配置不准确，或游戏画面中金条数字被遮挡/模糊。请使用坐标工具重新校准 `gold_region`，尽量只框住数字部分。
    -   `网络请求失败`: 通常是Web服务器没有启动。请检查启动器中“服务器”的状态。

**Q3: Web页面 (localhost:8080) 打不开？**
**A3:**
-   **检查服务器状态**: 确保启动器中“服务器”的状态为“运行中”。
-   **端口冲突**: 可能是 `8080` 端口被其他程序（如其他开发工具、下载软件等）占用了。请关闭占用程序后重试。

**Q4: 金条数字识别不准确？**
**A4:**
-   **重新校准坐标**: 这是最常见的原因。确保 `gold_region` 区域精确地框住了数字，背景尽量单一。
-   **游戏画质**: 尝试调高游戏内的画质和分辨率，使数字更清晰。
-   **图像预处理**: 对于高手，可以尝试修改 `scanner.py` 文件中的 `preprocess_image` 函数，调整图像处理参数（如对比度、锐化等）。

**Q5: 账号保存后，在分组页面变成了“未分配”？**
**A5:**
-   此问题在 v1.1 版本中已修复。后端增加了保护逻辑，在主面板保存时会自动忽略分组信息，防止其被错误覆盖。
